A simple, custom 8-bit virtual CPU emulator, complete with its own assembler and a GUI memory editor. This project was built as an educational stepping stone to understand the fundamentals of CPU emulation, the fetch-decode-execute cycle, and instruction sets before moving on to more complex systems like CHIP-8. 
## Project Files 
This repository contains three main files, each serving a distinct role in the emulation toolchain: 

 1. `cpu.c` (The Emulator) 
 
 This is the core virtual machine written in C. It implements the fetch-decode-execute cycle of a custom 8-bit CPU. 
 * **Memory:** 256 bytes of addressable memory. 
 * **Registers:** A Program Counter (`PC`) and an Accumulator (`ACC`). 
 * **Functionality:** It reads a hex dump file (generated by the assembler or memory editor) into its memory array and executes the instructions sequentially until it hits an `END` instruction. It outputs the CPU state and memory changes to the console. 
 
  1. `toy_assembler.py` (The Assembler) 
  
  A command-line Python script that translates human-readable assembly code into machine-readable hex code. 
  * **Functionality:** It features a simple lexer that reads an input text file containing custom assembly mnemonics (like `LDA`, `ADDA`, `GOTO`) and converts them into the corresponding operation codes (opcodes). 
  * **Output:** It generates a `.txt` file containing the compiled hex dump (one byte per line), which can then be fed directly into `cpu.c`
  
  1. `mem_editor.py` (The Memory Editor) A graphical utility built with Python and PyQt6 for directly manipulating CPU memory files. 
  * **Functionality:** It provides a grid view of all 256 memory addresses (`0x00` to `0xFF`). 
  * **Use Case:** You can use it to manually write machine code byte-by-byte, inspect the state of a memory dump saved after program execution, or tweak parameters of an assembled program without reassembling. It supports loading and saving the same hex dump format used by the CPU and assembler. 

## Architecture & Instruction Set (ISA)
The virtual CPU is an 8-bit accumulator machine with 256 bytes of addressable memory. 
### Instructions | Mnemonic | Opcode | Arguments | 
Description | | :--- | :---: | :--- | :--- | | 
- **LDA** | `0x01` | `[addr]` | Load the value at memory address `[addr]` into ACC. | |
- **LDI** | `0x02` | `[val]` | Load the immediate value `[val]` directly into ACC. | | 
- **ADDA** | `0x03` | `[addr]` | Add the value at memory address `[addr]` to ACC. | | 
- **ADDI** | `0x04` | `[val]` | Add the immediate value `[val]` to ACC. | |
- **STA** | `0x05` | `[addr]` | Store the current value of ACC into memory address `[addr] | | 
- **END** | `0x06` | *none* | Halt CPU execution. | | 
- **GOTO** | `0x07` | `[addr]` | Unconditionally jump PC to `[addr]`. | | 
- **BREQ** | `0x08` | `[val] [addr]` | Branch if equal: if ACC == `[val]`, jump PC to `[addr]` Otherwise, continue. |

##  Getting Started 

Prerequisites 
* **C Compiler:** (e.g., `gcc`) for compiling the emulator. 
* **Python 3.x:** For running the assembler and memory editor. 
* **PyQt6:** Required for the memory editor GUI. Install via: `pip install PyQt6` 

## Usage Workflow 
1. **Write an Assembly Program.** Create a text file (e.g., `test.asm`) and write your program using the supported mnemonics.

2. **Assemble the Code.** Use the assembler to convert your `.asm` file into a hex dump format.	

		Bash: $ python toy_assembler.py test.asm -o program.txt
		
3. **Run the Emulator.** Compile the C program and pass the generated hex file as an argument.

		Bash:  $  gcc cpu.c -o cpu
			 $ ./cpu program.txt
			 
4. **Edit Memory Manually (Optional)** Launch the GUI editor to inspect or manually create a memory state.

		Bash: $ python mem_editor.py
		
	From the GUI, you can load your `program.txt`, tweak memory values at specific addresses, and save them back.
